# Wix 网站迁移到香港服务器指南

## 当前情况

- **网站平台**: Wix
- **域名情况**: 
  - **域名 1**: 在 Wix 上注册和管理
  - **域名 2**: 在 Wix 上注册和管理
  - **域名 3**: 在 Amazon 上注册（可能是 Route 53）
- **目标**: 迁移到香港服务器
- **DNS 管理**: 
  - Wix 注册的域名：DNS 在 Wix 管理
  - Amazon 注册的域名：DNS 在 Amazon Route 53 管理

## 迁移方案概述

### 方案说明

**是的，可以！** 您只需要：
1. ✅ 在香港部署服务器和网站
2. ✅ 在 Onamae 修改 DNS 记录，指向香港服务器
3. ✅ 等待 DNS 传播（24-48 小时）

**重要**: 如果网站内容还在 Wix 上，您需要先：
- 从 Wix 导出网站内容
- 部署到香港服务器
- 然后修改 DNS 指向

## 详细步骤

### 第一步：准备香港服务器

#### 1.1 选择香港服务器
推荐服务商：
- **阿里云香港**：价格合理，网络稳定
- **腾讯云香港**：与国内连接好
- **AWS 香港**：稳定可靠
- **DigitalOcean 香港**：简单易用

#### 1.2 部署网站
将 Wix 网站内容部署到香港服务器：

**选项 A: 静态网站**
- 如果 Wix 网站是静态的，直接上传 HTML/CSS/JS 文件
- 使用 Nginx 或 Apache 作为 Web 服务器

**选项 B: 动态网站**
- 如果使用数据库，需要迁移数据库
- 配置 PHP/Node.js 等运行环境

### 第二步：修改 DNS 记录（根据域名注册位置）

根据域名的注册位置，需要在不同的平台修改 DNS：

#### 情况 A: Wix 注册的域名（2个域名）

##### 2.1 登录 Wix 管理后台
1. 访问 Wix 官网：https://www.wix.com
2. 登录您的账号
3. 进入网站管理后台

##### 2.2 进入域名设置
1. 点击左侧菜单的"设置"（Settings）
2. 选择"域名"（Domains）
3. 找到您要修改的域名（两个 Wix 注册的域名）
4. 点击"管理域名"（Manage Domain）或"DNS 设置"（DNS Settings）

##### 2.3 修改 DNS 记录（Wix）
1. **找到当前的 DNS 记录**
   - 查看指向 Wix 的 CNAME 或 A 记录

2. **修改 A 记录（主域名）**
   - 找到类型为 "A" 的记录（主机名为 @ 或留空）
   - 点击"编辑"（Edit）
   - 将值改为香港服务器的 IP 地址
   - 保存

3. **修改 A 记录（www 子域名）**
   - 找到主机名为 "www" 的 A 记录
   - 点击"编辑"（Edit）
   - 将值改为香港服务器的 IP 地址
   - 保存

4. **删除 Wix 的 CNAME 记录（如果有）**
   - 如果存在指向 `wix.com` 或 `wixdns.net` 的 CNAME 记录
   - 需要删除这些记录

#### 情况 B: Amazon 注册的域名（1个域名）

##### 2.4 登录 Amazon Route 53
1. 访问 AWS 控制台：https://console.aws.amazon.com
2. 登录您的 AWS 账号
3. 进入 Route 53 服务

##### 2.5 进入托管区域（Hosted Zones）
1. 在 Route 53 控制台中，点击"托管区域"（Hosted Zones）
2. 找到您要修改的域名
3. 点击域名进入记录集

##### 2.6 修改 DNS 记录（Route 53）
1. **修改 A 记录（主域名）**
   - 找到类型为 "A" 的记录（名称为 @ 或域名本身）
   - 点击"编辑"（Edit）
   - 将值改为香港服务器的 IP 地址
   - 保存

2. **修改 A 记录（www 子域名）**
   - 找到名称为 "www" 的 A 记录
   - 点击"编辑"（Edit）
   - 将值改为香港服务器的 IP 地址
   - 保存

3. **删除或修改其他记录**
   - 检查是否有指向 Wix 的 CNAME 记录
   - 根据需要删除或修改

##### 2.7 Route 53 操作步骤（详细）

**步骤 1: 进入 Route 53**
```
AWS 控制台 → Route 53 → 托管区域（Hosted Zones）
```

**步骤 2: 选择域名**
- 点击要修改的域名

**步骤 3: 编辑 A 记录**
- 找到类型为 "A" 的记录
- 点击记录名称进入编辑
- 修改值为香港服务器 IP
- 点击"保存更改"（Save changes）

**步骤 4: 处理其他记录**
- 检查 CNAME、MX 等其他记录
- 保留必要的记录（如 MX 邮件记录）
- 删除指向 Wix 的记录

#### 2.4 修改 DNS 记录

**在 Wix 的 DNS 设置中，需要修改以下记录：**

##### 方法 A: 使用 Wix 的 DNS 管理界面

1. **找到当前的 DNS 记录**
   - 查看指向 Wix 的 CNAME 或 A 记录

2. **修改 A 记录（主域名）**
   - 找到类型为 "A" 的记录（主机名为 @ 或留空）
   - 点击"编辑"（Edit）

   - 使用 Cloudflare、阿里云 DNS 等
   - 在 Wix 中修改名称服务器指向第三方 DNS

#### 2.5 Wix DNS 设置示例

**修改前（指向 Wix）:**
```
@    CNAME    wix.com
www  CNAME    wix.com
或
@    A    [Wix的IP地址]
www  A    [Wix的IP地址]
```

**修改后（指向香港服务器）:**
```
@    A    103.xxx.xxx.xxx    (香港服务器IP)
www  A    103.xxx.xxx.xxx    (香港服务器IP)
```

#### 2.6 Wix 界面操作步骤（详细）

**步骤 1: 进入域名设置**
```
Wix 后台 → 设置（Settings）→ 域名（Domains）
```

**步骤 2: 选择要修改的域名**
- 点击域名旁边的"管理"（Manage）或"DNS 设置"（DNS Settings）

**步骤 3: 查看当前 DNS 记录**
- 找到 "DNS 记录"（DNS Records）部分
- 查看当前的 A 记录和 CNAME 记录

**步骤 4: 编辑 A 记录**
- 点击 A 记录旁边的"编辑"（Edit）图标
- 修改 IP 地址为香港服务器 IP
- 点击"保存"（Save）

**步骤 5: 处理 CNAME 记录**
- 如果存在指向 Wix 的 CNAME 记录，需要删除或修改
- 注意：主域名（@）不能使用 CNAME，必须使用 A 记录

**步骤 6: 保存更改**
- 确认所有更改后，点击"保存"或"应用"（Apply）
- Wix 会提示 DNS 更改可能需要时间生效

### 第三步：配置服务器

#### 3.1 配置 Web 服务器（Nginx 示例）

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
```

#### 3.2 配置 SSL 证书（HTTPS）
使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
sudo apt-get update
sudo apt-get install certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

#### 3.3 配置防火墙
```bash
# 开放 80 和 443 端口
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw reload
```

### 第四步：等待 DNS 传播

#### 4.1 DNS 传播时间
- **通常**: 1-24 小时
- **最长**: 48 小时
- **TTL 设置**: 如果 TTL 设置较小（如 300），传播会更快

#### 4.2 检查 DNS 传播
使用以下工具检查：
- **全球检查**: https://www.whatsmydns.net
- **中国检查**: https://dnschecker.org
- **命令行**: `nslookup yourdomain.com`

#### 4.3 测试访问
```bash
# 测试域名解析
nslookup yourdomain.com

# 测试 HTTP 访问
curl -I http://yourdomain.com

# 测试 HTTPS 访问
curl -I https://yourdomain.com
```

## Wix DNS 配置详细说明（适用于域名 1 和 2）

### Wix 界面操作步骤（完整版）

#### 方法 1: 在 Wix 中直接修改 DNS（如果支持）

1. **登录 Wix**
   - 访问 https://www.wix.com
   - 登录您的账号

2. **进入域名设置**
   - 点击左侧菜单的"设置"（Settings）
   - 选择"域名"（Domains）
   - 找到您要修改的域名

3. **进入 DNS 管理**
   - 点击域名旁边的"管理"（Manage）
   - 或点击"DNS 设置"（DNS Settings）
   - 找到 "DNS 记录"（DNS Records）部分

4. **修改 A 记录**
   - 找到类型为 "A" 的记录（主机名为 @ 或 www）
   - 点击"编辑"（Edit）图标
   - 将 IP 地址改为香港服务器的 IP
   - 点击"保存"（Save）

5. **处理 CNAME 记录**
   - 如果存在指向 Wix 的 CNAME 记录
   - 需要删除或修改这些记录
   - 主域名（@）必须使用 A 记录，不能使用 CNAME

6. **保存更改**
   - 确认所有更改
   - 点击"保存"或"应用"（Apply）

#### 方法 2: 如果 Wix 不支持修改 DNS（使用名称服务器）

如果 Wix 的 DNS 管理功能受限，可以：

1. **在 Wix 中获取名称服务器信息**
   - 查看当前的名称服务器（Name Servers）
   - 记录这些信息

2. **在 Onamae 中修改名称服务器**
   - 登录 Onamae：https://www.onamae.com
   - 进入域名管理
   - 修改名称服务器为 Onamae 的 DNS 服务器
   - 或使用第三方 DNS 服务（如 Cloudflare）

3. **在新的 DNS 服务中配置记录**
   - 在 Onamae 或第三方 DNS 中添加 A 记录
   - 指向香港服务器 IP

### Wix DNS 记录类型说明

#### A 记录（IPv4）- 主要记录
```
类型: A
主机名: @ (或留空，表示主域名)
值: 103.xxx.xxx.xxx (香港服务器IP)
TTL: 3600 (或默认)
```

#### A 记录（www 子域名）
```
类型: A
主机名: www
值: 103.xxx.xxx.xxx (香港服务器IP)
TTL: 3600
```

#### 注意事项
- **主域名（@）必须使用 A 记录**，不能使用 CNAME
- **www 可以使用 A 记录或 CNAME**
- **保留 MX 记录**（如果使用企业邮箱）
- **保留其他必要的记录**（如 TXT 记录用于验证）

### 如果 Wix 限制 DNS 修改

某些情况下，Wix 可能限制 DNS 修改。解决方案：

#### 选项 A: 取消 Wix 域名连接
1. 在 Wix 中取消域名连接
2. 域名将回到注册商（Onamae）管理
3. 在 Onamae 中修改 DNS 记录

#### 选项 B: 使用第三方 DNS
1. 使用 Cloudflare、阿里云 DNS 等
2. 在 Wix 中修改名称服务器
3. 在第三方 DNS 中配置 A 记录

## 常见问题 FAQ

### Q1: 修改 DNS 后，网站会立即生效吗？
**A**: 不会立即生效。DNS 传播需要时间，通常 1-24 小时，最长 48 小时。

### Q2: DNS 传播期间，网站还能访问吗？
**A**: 
- 可以，但不同地区可能访问到不同的服务器
- 有些用户访问旧服务器（Wix），有些访问新服务器（香港）
- 这是正常现象

### Q3: 如何加快 DNS 传播？
**A**: 
- 设置较小的 TTL 值（如 300 秒）
- 等待当前 TTL 过期后，传播会更快
- 但无法完全控制传播速度

### Q4: 修改 DNS 会影响邮件吗？
**A**: 
- **不会**，只要不修改 MX 记录
- 只修改 A 记录和 CNAME 记录
- 邮件服务不受影响

### Q5: 如果修改错了怎么办？
**A**: 
- 在 Wix 中重新修改 DNS 记录
- 恢复到正确的配置
- 等待 DNS 传播（可能需要一些时间）

### Q6: 需要通知 Wix 吗？
**A**: 
- 如果只是修改 DNS 指向，不需要通知 Wix
- Wix 会自动检测到 DNS 更改
- 如果完全迁移，可以在 Wix 中取消域名连接

### Q7: Wix 不允许修改 DNS 怎么办？
**A**: 
- 选项 1: 在 Wix 中取消域名连接，回到注册商管理
- 选项 2: 使用第三方 DNS 服务（如 Cloudflare），修改名称服务器
- 选项 3: 联系 Wix 客服，询问如何修改 DNS 记录

### Q8: 修改 DNS 后，Wix 网站还能访问吗？
**A**: 
- 如果网站内容还在 Wix 上，修改 DNS 后网站将无法从 Wix 访问
- **必须先将网站内容迁移到香港服务器**
- 然后修改 DNS 指向新服务器

### Q11: 三个域名需要分别修改吗？
**A**: 
- **是的**，需要分别修改
- Wix 注册的 2 个域名：在 Wix 后台修改
- Amazon 注册的 1 个域名：在 AWS Route 53 中修改
- 所有域名都指向同一个香港服务器 IP

### Q12: 三个域名可以指向不同的服务器吗？
**A**: 
- **可以**，但通常建议指向同一个服务器
- 如果指向不同服务器，需要分别配置
- 如果指向同一服务器，可以共享配置

### Q13: Amazon Route 53 修改 DNS 需要多久生效？
**A**: 
- Route 53 的 DNS 传播通常很快
- 通常在几分钟到 1 小时内生效
- 比 Wix 的 DNS 传播更快

### Q9: 香港服务器需要什么配置？
**A**: 
- 至少 1GB 内存
- 支持 HTTPS（SSL 证书）
- 配置防火墙开放 80/443 端口
- 安装 Web 服务器（Nginx 或 Apache）

### Q10: 可以同时指向 Wix 和香港服务器吗？
**A**: 
- **不建议**，会导致访问不一致
- 应该完全迁移到香港服务器
- 或者使用负载均衡（高级方案）

## 总结

### ✅ 可以这样做
1. 在香港部署服务器和网站
2. **在 Wix 修改 DNS 记录**（A 记录）
3. 指向香港服务器的 IP 地址
4. 等待 DNS 传播完成

### 📋 关键步骤
1. **准备**: 部署网站到香港服务器
2. **修改 DNS**: 
   - **Wix 注册的域名**（2个）：在 Wix 后台修改 A 记录
   - **Amazon 注册的域名**（1个）：在 AWS Route 53 中修改 A 记录
3. **等待**: DNS 传播 1-48 小时（Route 53 通常更快）
4. **测试**: 确认所有域名正常访问

### ⚠️ 注意事项
- **必须先迁移网站内容**到香港服务器，再修改 DNS
- **三个域名需要分别修改**，但都指向同一个香港服务器 IP
- 不要删除 MX 记录（邮件服务）
- 备份修改前的 DNS 配置（三个域名都要备份）
- 测试服务器后再修改 DNS
- 等待 DNS 传播完成
- Route 53 的 DNS 传播通常比 Wix 更快
- 如果 Wix 不允许修改 DNS，考虑使用第三方 DNS 服务
